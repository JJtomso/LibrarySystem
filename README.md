# 📚 图书馆管理系统

一个轻量级的图书馆管理 Web 应用，提供图书浏览、借阅、管理等功能，支持**学生**与**管理员**独立登录，并内置密码保护与首次强制修改密码机制。后端使用 C + libmicrohttpd，前端原生 HTML/CSS/JavaScript，数据持久化至二进制文件。

---

## ✨ 功能特性

- **双角色登录**：学生与管理员使用统一登录入口，身份自动识别。
- **密码保护**：
  - 学生注册时默认密码 `123456`，首次登录**强制修改密码**。
  - 管理员默认账号 `admin` / `Admin@123`，可随时修改密码。
- **图书浏览**：展示所有图书的编号、书名、作者、出版社、单价、存放地点、库存状态。
- **借阅功能**（仅学生）：
  - 输入借阅天数（默认 30 天），自动计算应还日期。
  - 同一本书未归还前不可重复借阅。
  - 可查看个人借阅历史及逾期天数。
- **管理员后台**：
  - 录入新图书（若编号已存在则增加馆藏数量）。
  - 删除图书（仅当全部副本已归还）。
  - 调整馆藏数量（增加/减少副本）。
  - 查看所有逾期记录（含读者姓名和逾期天数）。
- **数据持久化**：所有数据（读者、图书、借阅记录）保存至本地二进制文件 `library.dat`。

## 🚀 快速开始

### 1. 安装依赖

**后端依赖**：libmicrohttpd（Ubuntu/Debian）
```bash
sudo apt-get install libmicrohttpd-dev
```
其他 Linux 发行版请使用对应包管理器安装。

### 2. 编译后端

```bash
gcc -o library_server server.c -lmicrohttpd -lpthread
```

### 3. 启动后端服务

```bash
./library_server
```
服务默认监听 `8888` 端口，启动后会打印日志并自动创建 `library.dat` 及样例数据（若文件不存在）。

### 4. 部署前端

前端为纯静态页面，可直接使用任意 HTTP 服务器（如 Python 简易服务器）：
```bash
python3 -m http.server 8080
```
或使用 VS Code Live Server 等工具。

**注意**：前端代码中 `API = 'http://localhost:8888'`，请确保后端地址与前端访问地址一致（如跨域需配置 CORS，后端已内置支持）。

### 5. 访问系统

在浏览器中打开 `http://localhost:8080`（或您指定的端口）即可进入登录页。

| 方法 | 端点                 | 说明                                   | 请求体示例                                         |
|------|----------------------|----------------------------------------|----------------------------------------------------|
| POST | `/login`             | 登录（支持学生/管理员）                | `{"username":"202100101","password":"123456"}`     |
| POST | `/register`          | 学生注册（密码自动设为123456）         | `{"id":"202100102","name":"李四","gender":"男"}`   |
| POST | `/change-password`   | 修改密码（学生或管理员）               | `{"username":"202100101","old_password":"123456","new_password":"new123"}` |
| GET  | `/books`             | 获取所有图书信息                       | -                                                  |
| POST | `/borrow`            | 学生借书                               | `{"reader_id":"202100101","book_id":"TP312C","days":30}` |
| GET  | `/myrecords?reader_id=xxx` | 查询个人借阅记录（含逾期天数）         | -                                                  |
| GET  | `/overdue`           | 获取所有逾期记录（管理员）             | -                                                  |
| POST | `/add-book`          | 录入新图书或增加副本（管理员）         | `{"id":"TP123","name":"新书","author":"作者",...}` |

## 🧑‍💻 使用说明

### 学生用户

1. **注册**：点击“注册新账号”，填写学号（9位）、姓名、性别，提交后默认密码为 `123456`。
2. **登录**：输入学号和密码，首次登录会跳转至修改密码页，需设置新密码（至少6位）。
3. **浏览图书**：登录后自动进入图书列表，可查看所有图书及状态。
4. **借书**：点击图书对应的“借书”按钮，输入借阅天数，系统自动计算应还日期。
5. **查看记录**：点击导航栏“我的借阅”，查看历史借阅及逾期情况。

### 管理员用户

1. **登录**：使用 `admin` / `Admin@123` 登录（首次无需改密）。
2. **管理图书**：
   - 录入新图书：填写完整信息，若编号已存在则增加库存。
   - 删除图书：输入图书编号，若全部副本已归还则可删除。
   - 调整数量：输入正数增加副本，负数减少副本（不能少于已借出数）。
3. **监控逾期**：在管理员页面底部查看所有逾期记录。
4. **修改密码**：在管理员页面底部可修改管理员密码。

## ⚙️ 配置与扩展

- **端口修改**：编辑 `server.c` 中的 `#define PORT 8888` 并重新编译。
- **数据文件路径**：修改 `DATA_FILE "library.dat"` 可更改存储位置。
- **最大容量**：`MAX_READERS`, `MAX_BOOKS`, `MAX_RECORDS` 可根据需要调整。
- **前端 API 地址**：修改 `script.js` 中的 `API` 变量。

## 🔧 常见问题

1. **启动后端时提示端口被占用**  
   检查是否已有 `library_server` 进程运行，或修改端口。

2. **浏览器控制台出现 CORS 错误**  
   后端已添加 `Access-Control-Allow-Origin: *`，若仍出现请确保前后端地址正确，或使用同一域名/端口。

3. **修改密码后仍跳回改密页**  
   请检查后端日志，确认 `need_change` 已置为 `0` 且数据保存成功。若问题依旧，可尝试清理浏览器缓存。

4. **借书失败提示“already borrowed and not returned”**  
   表明您已借阅该图书且尚未归还，请先归还（当前系统暂不提供归还功能，可通过数据库直接修改记录，或联系管理员处理）。
